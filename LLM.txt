# commandapi-kotlin-improved — LLM Reference Document

## Overview

`commandapi-kotlin-improved` is a Kotlin DSL extension library that improves upon the official Kotlin DSL
bundled with [CommandAPI](https://github.com/JorelAli/CommandAPI), a command registration library for
Minecraft plugins. It is a drop-in replacement for the official DSL (`dev.jorel.commandapi.kotlindsl`)
and its primary improvement is **type-safe argument value retrieval**.

- **Repository**: https://github.com/sya-ri/commandapi-kotlin-improved
- **Supported CommandAPI version**: 11.1.0
- **Kotlin version**: 2.3.0
- **Java version**: 17
- **License**: MIT
- **Author**: sya-ri (contact@s7a.dev)
- **Maven Group ID**: `dev.s7a`

## Supported Platforms and Artifacts

| Platform | Artifact ID | Target |
|---|---|---|
| Bukkit | `commandapi-bukkit-kotlin-improved` | Bukkit / CraftBukkit |
| Paper | `commandapi-paper-kotlin-improved` | Paper (superset of Bukkit) |
| Spigot | `commandapi-spigot-kotlin-improved` | Spigot (superset of Bukkit + Spigot-specific arguments) |
| Velocity | `commandapi-velocity-kotlin-improved` | Velocity proxy server |

Paper and Spigot include all Bukkit arguments plus their own extras.
Velocity is an independent implementation without Minecraft-specific arguments.

### Notes

- This library depends on CommandAPI itself. You still need to set up CommandAPI separately
  (add it to `plugin.yml` as a dependency, or shade it into your plugin JAR).
- With `compileOnly`, the CommandAPI JAR must be present at runtime. If you are shading the library
  into your plugin, use `implementation` with the appropriate shadow/relocation configuration.

## CommandAPI Initialization

CommandAPI must be initialized before registering commands. Example for Paper:

```kotlin
import dev.jorel.commandapi.CommandAPI
import dev.jorel.commandapi.CommandAPIPaperConfig
import org.bukkit.plugin.java.JavaPlugin

class MyPlugin : JavaPlugin() {
    override fun onLoad() {
        CommandAPI.onLoad(CommandAPIPaperConfig(this).verboseOutput(true))
        // Register commands here, not in onEnable
        registerCommands()
    }

    override fun onEnable() {
        CommandAPI.onEnable()
    }

    override fun onDisable() {
        CommandAPI.onDisable()
    }
}
```

## Difference from the Official DSL

### Official DSL — unsafe cast

```kotlin
// Package: dev.jorel.commandapi.kotlindsl
import dev.jorel.commandapi.kotlindsl.*

commandTree("example") {
    literalArgument("integer") {
        integerArgument("value") {
            anyExecutor { sender, args ->
                val value = args.get("value") as Int  // dangerous cast
                sender.sendMessage("You entered: $value")
            }
        }
    }
}
```

### Improved DSL — type-safe getter

```kotlin
// Package: dev.s7a.commandapi
import dev.s7a.commandapi.*

commandTree("example") {
    literalArgument("integer") {
        integerArgument("value") { getValue ->  // getValue: (CommandArguments) -> Int
            anyExecutor { sender, args ->
                val value = getValue(args)  // returns Int, no cast needed
                sender.sendMessage("You entered: $value")
            }
        }
    }
}
```

**Key improvements**:
- Each argument function's lambda receives a getter `(CommandArguments) -> T` as its parameter.
- Calling the getter inside an executor returns the correctly typed value without casting.
- Optional arguments provide `(CommandArguments) -> T?` (nullable).
- Type errors are caught at compile time — no `ClassCastException` at runtime.

## Basic Usage

### Imports

Either wildcard or explicit imports are fine.

```kotlin
// Wildcard import
import dev.s7a.commandapi.*

// Explicit imports (if you prefer)
import dev.s7a.commandapi.commandTree
import dev.s7a.commandapi.literalArgument
import dev.s7a.commandapi.integerArgument
import dev.s7a.commandapi.anyExecutor
```

### commandTree — Registering a Command

```kotlin
// Basic
commandTree("mycommand") {
    // add subcommands or arguments here
}

// With a string namespace
commandTree("mycommand", "myplugin") {
    // accessible as both /mycommand and /myplugin:mycommand
}

// With a JavaPlugin namespace
commandTree("mycommand", this) {
    // namespace is derived from the plugin name
}
```

## Arguments DSL

All argument functions are extension functions on both `CommandTree` and `Argument<*>`,
so you can nest them freely at any level.

### Common Pattern

```kotlin
// Required argument
someArgument("nodeName") { getValue ->
    // getValue: (CommandArguments) -> T
    anyExecutor { sender, args ->
        val v = getValue(args)  // T
    }
}

// Optional argument (xxxOptionalArgument)
someOptionalArgument("nodeName") { getValue ->
    // getValue: (CommandArguments) -> T?
    anyExecutor { sender, args ->
        val v = getValue(args)  // T? — may be null
    }
}
```

### Numeric Arguments

```kotlin
// Int — defaults: min = Int.MIN_VALUE, max = Int.MAX_VALUE
integerArgument("count", min = 1, max = 64) { getCount ->
    anyExecutor { sender, args ->
        val count: Int = getCount(args)
    }
}

// Float
floatArgument("speed", min = 0.0f, max = 10.0f) { getSpeed ->
    anyExecutor { sender, args ->
        val speed: Float = getSpeed(args)
    }
}

// Double
doubleArgument("ratio") { getRatio ->
    anyExecutor { sender, args ->
        val ratio: Double = getRatio(args)
    }
}

// Long
longArgument("timestamp") { getTimestamp ->
    anyExecutor { sender, args ->
        val ts: Long = getTimestamp(args)
    }
}

// IntegerRange — accepts input like "1..10" or "5.."
integerRangeArgument("range") { getRange ->
    anyExecutor { sender, args ->
        val range: IntegerRange = getRange(args)
        // range.lowerBound, range.upperBound
    }
}

// DoubleRange
doubleRangeArgument("range") { getRange ->
    anyExecutor { sender, args ->
        val range: DoubleRange = getRange(args)
    }
}
```

### Generic / Custom Arguments

Use `argument` / `optionalArgument` directly when the type is not covered by a built-in helper,
or when you want to wrap a `CustomArgument` on an ad-hoc basis.

```kotlin
// On CommandTree
commandTree("mycommand") {
    argument<MyType, MySubType>(MyCustomArgument("argName")) { getValue ->
        anyExecutor { sender, args ->
            val v: MySubType = getValue(args)
        }
    }
}

// On Argument (nested)
literalArgument("sub") {
    argument<MyType, MySubType>(MyCustomArgument("argName")) { getValue ->
        anyExecutor { sender, args ->
            val v: MySubType = getValue(args)
        }
    }
}

// Optional variant
optionalArgument<MyType, MySubType>(MyCustomArgument("argName")) { getValue ->
    anyExecutor { sender, args ->
        val v: MySubType? = getValue(args)
    }
}
```

### Creating a First-Class Custom Argument Function

To call your custom argument the same way as the built-in ones — e.g. `speedModeArgument("mode") { getMode -> ... }` —
define it as `inline` extension functions on both `CommandTree` and `Argument<*>`, mirroring the pattern the library uses internally.

```kotlin
import dev.jorel.commandapi.CommandTree
import dev.jorel.commandapi.arguments.Argument
import dev.jorel.commandapi.arguments.CustomArgument
import dev.jorel.commandapi.arguments.CustomArgument.CustomArgumentException
import dev.jorel.commandapi.arguments.MultiLiteralArgument
import dev.jorel.commandapi.executors.CommandArguments

enum class SpeedMode { WALK, FLY, SWIM }

@Suppress("FunctionName")
fun SpeedModeArgument(nodeName: String) =
    CustomArgument(
        MultiLiteralArgument(nodeName, *SpeedMode.entries.map { it.name.lowercase() }.toTypedArray())
    ) { info ->
        SpeedMode.entries.firstOrNull { it.name.equals(info.input(), ignoreCase = true) }
            ?: throw CustomArgumentException.fromString("Unknown mode: ${info.input()}")
    }

// Extension on CommandTree — usable at the root level
inline fun CommandTree.speedModeArgument(
    nodeName: String,
    crossinline block: Argument<*>.((CommandArguments) -> SpeedMode) -> Unit = {},
) = argument(SpeedModeArgument(nodeName), block)

// Extension on Argument<*> — usable when nested inside another argument
inline fun Argument<*>.speedModeArgument(
    nodeName: String,
    crossinline block: Argument<*>.((CommandArguments) -> SpeedMode) -> Unit = {},
) = argument(SpeedModeArgument(nodeName), block)
```

Usage is now identical to any built-in argument:

```kotlin
commandTree("speed") {
    speedModeArgument("mode") { getMode ->
        floatArgument("value", min = 0.0f, max = 1.0f) { getValue ->
            playerExecutor { sender, args ->
                val mode: SpeedMode = getMode(args)   // SpeedMode, not String
                val value = getValue(args)
                when (mode) {
                    SpeedMode.WALK -> sender.walkSpeed = value
                    SpeedMode.FLY  -> sender.flySpeed  = value
                    SpeedMode.SWIM -> { /* handle swim speed */ }
                }
                sender.sendMessage("Set ${mode.name.lowercase()} speed to $value")
            }
        }
    }
}
```

The same pattern works for any type — wrap whatever `Argument<T>` you need in the raw factory function
and expose it through `CommandTree` and `Argument<*>` extensions.

## Executors DSL

Executor functions are extension functions on `BukkitExecutable<*>` (or `VelocityExecutable<*>` on Velocity).
Call them inside an argument block.

### Bukkit / Paper / Spigot

#### Standard Executors (no return value)

```kotlin
// Any sender
anyExecutor { sender: CommandSender, args: CommandArguments ->
    sender.sendMessage("Hello!")
}

// Players only — non-player senders cannot run the command
playerExecutor { sender: Player, args: CommandArguments ->
    sender.sendMessage("Hi, ${sender.name}!")
}

// Entities only
entityExecutor { sender: Entity, args: CommandArguments -> }

// Console only
consoleExecutor { sender: ConsoleCommandSender, args: CommandArguments -> }

// Command block only
commandBlockExecutor { sender: BlockCommandSender, args: CommandArguments -> }

// Proxied sender (e.g. "execute ... run")
proxyExecutor { sender: ProxiedCommandSender, args: CommandArguments -> }

// Native proxy
nativeExecutor { sender: NativeProxyCommandSender, args: CommandArguments -> }

// Remote console (RCON)
remoteConsoleExecutor { sender: RemoteConsoleCommandSender, args: CommandArguments -> }
```

#### Resulting Executors (return Int)

Return a result code for the command. `0` means failure; a positive value means success.

```kotlin
anyResultingExecutor { sender: CommandSender, args: CommandArguments -> 1 }
playerResultingExecutor { sender: Player, args: CommandArguments -> if (sender.isOp) 1 else 0 }

// Also available: entityResultingExecutor, consoleResultingExecutor,
// commandBlockResultingExecutor, proxyResultingExecutor,
// nativeResultingExecutor, remoteConsoleResultingExecutor
```

#### ExecutionInfo Executors

Provides a richer `ExecutionInfo` context (access to the command node, etc.).

```kotlin
anyExecutionInfo { info: ExecutionInfo<CommandSender, BukkitCommandSender<out CommandSender>> ->
    val sender = info.sender()
    val args   = info.args()
    sender.sendMessage("Executed!")
}

playerExecutionInfo { info: ExecutionInfo<Player, BukkitPlayer> ->
    info.sender().sendMessage("Player: ${info.sender().name}")
}

// Resulting variants
anyResultingExecutionInfo { info -> 1 }
playerResultingExecutionInfo { info -> 1 }

// Also available for every sender type:
// entityExecutionInfo, consoleExecutionInfo, commandBlockExecutionInfo,
// proxyExecutionInfo, nativeExecutionInfo, remoteConsoleExecutionInfo
// and their Resulting counterparts
```

### Velocity

```kotlin
anyExecutor { sender: CommandSource, args: CommandArguments ->
    sender.sendMessage(Component.text("Hello!"))
}

playerExecutor { sender: Player, args: CommandArguments ->
    sender.sendMessage(Component.text("Hi, ${sender.username}!"))
}

consoleExecutor { sender: ConsoleCommandSource, args: CommandArguments -> }

// Resulting variants
anyResultingExecutor    { sender, args -> 1 }
playerResultingExecutor { sender, args -> 1 }
consoleResultingExecutor { sender, args -> 1 }

// ExecutionInfo variants
anyExecutionInfo    { info: ExecutionInfo<CommandSource, VelocityCommandSender<out CommandSource>> -> }
playerExecutionInfo { info: ExecutionInfo<Player, VelocityPlayer> -> }
consoleExecutionInfo { info: ExecutionInfo<ConsoleCommandSource, VelocityConsoleCommandSender> -> }

// Resulting ExecutionInfo variants
anyResultingExecutionInfo    { info -> 1 }
playerResultingExecutionInfo { info -> 1 }
consoleResultingExecutionInfo { info -> 1 }
```

## Nesting (Subcommands)

Arguments can be nested to any depth. Use `literalArgument` for subcommand keywords.

```kotlin
commandTree("admin") {
    // /admin reload
    literalArgument("reload") {
        anyExecutor { sender, args ->
            sender.sendMessage("Reloaded!")
        }
    }

    // /admin set <player> <value>
    literalArgument("set") {
        entitySelectorArgumentOnePlayer("player") { getPlayer ->
            integerArgument("value", min = 0, max = 100) { getValue ->
                anyExecutor { sender, args ->
                    val player = getPlayer(args)
                    val value  = getValue(args)
                    sender.sendMessage("Set ${player.name}'s value to $value")
                }
            }
        }
    }

    // /admin info [player]  ← optional
    literalArgument("info") {
        entitySelectorOptionalArgumentOnePlayer("player") { getPlayer ->
            anyExecutor { sender, args ->
                val player: Player? = getPlayer(args)
                sender.sendMessage(
                    if (player != null) "Info for: ${player.name}" else "No player specified"
                )
            }
        }
    }
}
```

## Optional Arguments

Use `xxxOptionalArgument` to make an argument skippable. The getter returns `T?`.

```kotlin
commandTree("greet") {
    stringOptionalArgument("name") { getName ->
        anyExecutor { sender, args ->
            val name: String? = getName(args)
            sender.sendMessage(if (name != null) "Hello, $name!" else "Hello, world!")
        }
    }
}
```

## Practical Examples

### Example 1: Custom argument wrapping an enum

Defining a first-class DSL function (see "Creating a First-Class Custom Argument Function" above)
lets the call site look identical to any built-in argument. The enum conversion and error handling
live in one place; executors just receive `SpeedMode` directly.

```kotlin
// Definition (typically in a shared file)
enum class SpeedMode { WALK, FLY, SWIM }

@Suppress("FunctionName")
fun SpeedModeArgument(nodeName: String) =
    CustomArgument(
        MultiLiteralArgument(nodeName, *SpeedMode.entries.map { it.name.lowercase() }.toTypedArray())
    ) { info ->
        SpeedMode.entries.firstOrNull { it.name.equals(info.input(), ignoreCase = true) }
            ?: throw CustomArgument.CustomArgumentException.fromString("Unknown mode: ${info.input()}")
    }

inline fun CommandTree.speedModeArgument(
    nodeName: String,
    crossinline block: Argument<*>.((CommandArguments) -> SpeedMode) -> Unit = {},
) = argument(SpeedModeArgument(nodeName), block)

inline fun Argument<*>.speedModeArgument(
    nodeName: String,
    crossinline block: Argument<*>.((CommandArguments) -> SpeedMode) -> Unit = {},
) = argument(SpeedModeArgument(nodeName), block)

// Usage — indistinguishable from a built-in argument
commandTree("speed") {
    speedModeArgument("mode") { getMode ->
        floatArgument("value", min = 0.0f, max = 1.0f) { getValue ->
            playerExecutor { sender, args ->
                val mode: SpeedMode = getMode(args)
                val value = getValue(args)
                when (mode) {
                    SpeedMode.WALK -> sender.walkSpeed = value
                    SpeedMode.FLY  -> sender.flySpeed  = value
                    SpeedMode.SWIM -> { /* handle swim speed */ }
                }
                sender.sendMessage("Set ${mode.name.lowercase()} speed to $value")
            }
        }
    }
}
```

### Example 2: Entity selector and item giving

```kotlin
commandTree("give") {
    entitySelectorArgumentOnePlayer("target") { getTarget ->

            }
        }
    }
}
```

## Internal Implementation Notes

These notes are provided so AI code generators understand the design and can extend or replicate patterns correctly.

### How `argument` / `optionalArgument` work

```kotlin
// Generic implementation on CommandTree
inline fun <reified T, reified U : T> CommandTree.argument(
    base: Argument<T>,
    crossinline block: Argument<*>.((CommandArguments) -> U) -> Unit = {},
): CommandTree =
    then(
        base.apply {
            this.block { args ->
                args.get(base.nodeName) as U  // cast is encapsulated here
            }
        },
    )

// Each specific argument type is a thin wrapper
inline fun CommandTree.integerArgument(
    nodeName: String,
    min: Int = Int.MIN_VALUE,
    max: Int = Int.MAX_VALUE,
    crossinline block: Argument<*>.((CommandArguments) -> Int) -> Unit = {},
) = argument(IntegerArgument(nodeName, min, max), block)
```

The `block` parameter type `Argument<*>.((CommandArguments) -> U) -> Unit` is a lambda with
`Argument<*>` as receiver and a getter `(CommandArguments) -> U` as its first argument.
The getter is constructed once and closed over by the executor lambda.

### How executors work

```kotlin
inline fun BukkitExecutable<*>.anyExecutor(
    crossinline executor: (CommandSender, CommandArguments) -> Unit,
): BukkitExecutable<*> =
    executes(CommandExecutor { sender, args -> executor(sender, args) })
```

All executor functions use `inline` + `crossinline`, so no lambda object is allocated at runtime.

## Patterns and Anti-Patterns

### Correct — use the getter

```kotlin
integerArgument("count") { getCount ->
    anyExecutor { sender, args ->
        val count = getCount(args)  // type-safe
    }
}
```

### Avoid — manual cast (official DSL style, unnecessary here)

```kotlin
integerArgument("count") { _ ->
    anyExecutor { sender, args ->
        val count = args.get("count") as Int  // works but defeats the purpose
    }
}
```

### Avoid — calling the getter outside an executor

```kotlin
integerArgument("count") { getCount ->
    // val count = getCount(???)  // no args available here
    anyExecutor { sender, args ->
        val count = getCount(args)  // correct: call inside the executor
    }
}
```
